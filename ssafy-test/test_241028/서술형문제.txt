문제 : 디스패처 서블릿(DispatcherServlet)은 Spring MVC에서 핵심적인 역할을 합니다. MVC 구조에서 클라이언트의 요청이 디스패처 서블릿을 통해 처리되는 과정을 설명하세요. 

1. 디스패처 서블릿이 클라이언트의 요청을 받는다 -> 핸들러컨트롤러 요청
2. 핸들러컨트롤러에서 요청을 처리할 컨트롤러를 찾는다 -> 디스패처 서블릿에 반환
3. 디스패처 서블릿에서는 다시 해당 컨트롤러로 요청을 보낸다
4. 컨트롤러에서는 -Service-(dto)-dao-DB 과정을 통해 ModelAndView 객체를 반환한다
*dto: data-transfer-object : 데이터를 처리하는데 사용되는 객체
*dao : dao: data-access-object : DB 와의 연결에 사용되는 객체
5. 디스패처 서블릿에서는 뷰 리졸버로 요청을 보낸다
6. 뷰 리졸버에서는 해당 뷰를 처리할 수 있는 뷰를 찾는다 -> 디스패처 서블릿에 반환
7. 디스패처 서블릿에서는 해당 뷰와 모델을 클라이언트에 반환한다.
